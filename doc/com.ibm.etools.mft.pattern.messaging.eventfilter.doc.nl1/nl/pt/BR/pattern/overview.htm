Padrão Filtro de Evento de Entrada do IBM MessageSight

Este padrão usa o IBM Integration Bus para receber eventos de clientes móveis via IBM MessageSight, filtrar os eventos com base em critérios especificados e encaminhar eventos correspondentes aos critérios para um sistema backend.

Use este padrão quando as condições a seguir forem verdadeiras:

  • Grandes números de clientes móveis estiverem publicando mensagens via IBM MessageSight.
  • Apenas um subconjunto de mensagens publicadas requererem processamento por um sistema backend.

Solução

Se um volume alto de clientes publicar mensagens via IBM MessageSight, a ativação do processamento de negócios para cada mensagem poderá sobrecarregar os sistemas backend. Com o uso do padrão de Filtro de Evento de Entrada do IBM MessageSight, apenas as mensagens que correspondem aos critérios especificados são encaminhadas para os sistemas backend para processamento de negócios. As mensagens que não correspondem aos critérios podem ser registradas ou descartadas.

Por exemplo, dispositivos médicos (como monitores de marca-passo) podem publicar mensagens de evento continuamente sobre o status do paciente e do dispositivo. As mensagens de evento que contêm atualizações de status regulares podem ser registradas e as que contêm alertas de severidade alta podem ser encaminhadas para os sistemas de processamento de negócios.
