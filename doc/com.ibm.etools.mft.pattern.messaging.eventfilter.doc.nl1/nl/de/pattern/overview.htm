Muster 'IBM MessageSight Inbound Event Filter'

Dieses Muster verwendet IBM Integration Bus für den Empfang von Ereignissen aus mobilen Clients über IBM MessageSight sowie zum Filtern der Ereignisse auf Basis bestimmter Kriterien und zur Weiterleitung von Ereignissen, die den Kriterien entsprechen, an ein Back-End-System.

Verwenden Sie dieses Muster, wenn die folgenden Bedingungen zutreffen:

  • Sehr viele mobile Clients veröffentlichen Nachrichten über IBM MessageSight.
  • Nur eine Teilmenge der veröffentlichten Nachrichten erfordert die Verarbeitung durch ein Back-End-System.

Lösung

Wenn sehr viele Clients Nachrichten über IBM MessageSight veröffentlichen, besteht die Gefahr, dass die Back-End-Systeme durch die Aktivierung der Geschäftsprozessverarbeitung für jede Nachricht überlastet werden. Bei Verwendung des Musters 'IBM MessageSight Inbound Event Filter' werden nur Nachrichten, die bestimmten Kriterien entsprechen, an die Back-End-Systeme für die Geschäftsprozessverarbeitung weitergeleitet. Nachrichten, die die Kriterien nicht erfüllen, können protokolliert oder verworfen werden.

Manche medizinische Geräte (zum Beispiel Überwachungsgeräte von Schrittmachern) veröffentlichen möglicherweise fortlaufend Ereignisnachrichten zum Zustand des Patienten und des Geräts. Ereignisnachrichten, die regelmäßige Statusaktualisierungen enthalten, können protokolliert werden, während Nachrichten mit Alerts hoher Dringlichkeit an die Geschäftsprozesssysteme weitergeleitet werden können.
