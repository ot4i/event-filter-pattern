Tasks, die nach der Generierung des Musters IBM MessageSight Inbound Event Filter ausgeführt werden müssen

Tasks, die Sie nach der Generierung des Musters ausführen müssen.

Führen Sie nach der Generierung der Musterinstanz 'IBM MessageSight Inbound Event Filter' die folgenden Schritte aus:

  1. Öffnen Sie den Nachrichtenfluss 'eventFilter' und ermitteln Sie die generierten Befehle, die Sie für die Konfiguration des IBM MessageSight-Endpunkts ausführen müssen. Führen Sie diese Befehle auf Ihrer IBM MessageSight-Maschine aus.
  2. Optional: Wenn Sie die generierte Anwendung auf einem fernen Integrationsknoten (Broker) ausführen möchten, führen Sie die folgenden Schritte aus:
    1. Kopieren Sie unter Verwendung der Navigatoransicht die generierte Datei mit der Erweiterung .bindings aus dem Projekt Musterinstanzname_EventFilter auf Ihre IBM Integration Bus-Maschine.
    2. Navigieren Sie im generierten Nachrichtenfluss zu den Eigenschaften des JMS-Empfangsknotens 'Event Subscriber'. Aktualisieren Sie auf der Registerkarte für die JMS-Verbindung die Eigenschaft Location JNDI Bindings (Speicherposition der JNDI-Bindungen) mit der Speicherposition der Datei mit der Erweiterung .bindings.
  3. Erstellen Sie den Warteschlangenmanager, die von Ihnen für gefilterte Ereignisse angegebene Warteschlange und die Warteschlange, die Sie für die Fehlerbehandlung angegeben haben (falls diese noch nicht vorhanden ist).
  4. Implementieren Sie die generierte Anwendung Musterinstanzname_EventFilter auf Ihrem Integrationsknoten.

Wenn Sie bei der Konfiguration der Musterinstanz die Standardwerte für die Parameter akzeptiert haben, können Sie die generierte Anwendung mit folgenden Schritten testen:

  1. Öffnen Sie im Navigator der Anwendungsentwicklung über Independent Resources > YourPatternInstanceName_EventFilter_client > PacemakerClient.html (Unabhängige Ressourcen > Musterinstanzname_EventFilter_client PacemakerClient.html) die Datei in einem HTML5-fähigen Browser.
  2. Klicken Sie auf der Browserseite auf Connect (Verbinden), um eine Verbindung zum IBM MessageSight-Endpunkt herzustellen.
  3. Senden Sie eine Alertereignisnachricht an IBM Integration Bus, indem Sie im Feld Event Type (Ereignistyp) den Eintrag Alert auswählen und auf Publish (Veröffentlichen) klicken.
  4. Wiederholen Sie den obigen Schritt mehrmals, wählen Sie dabei jedoch im Feld Event Type (Ereignistyp) andere Werte aus.
  5. Überprüfen Sie die MQ-Warteschlange, um sicherzustellen, dass sich die Alertereignisnachrichten (und nur die Alertereignisnachrichten) in der Warteschlange befinden.
