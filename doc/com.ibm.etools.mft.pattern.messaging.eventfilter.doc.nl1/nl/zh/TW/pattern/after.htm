產生 IBM MessageSight Inbound Event Filter 型樣之後要完成的作業

在您產生此型樣之後必須完成的作業。

產生 IBM MessageSight Inbound Event Filter 型樣實例之後，請完成下列步驟：

  1. 開啟 eventFilter 訊息流程，確定產生的指令，這些指令必須執行用來配置 IBM MessageSight 端點。在 IBM MessageSight 機器上執行這些指令。
  2. 選用項目：如果要在遠端整合節點（分配管理系統）上執行產生的應用程式，請完成下列步驟：
    1. 使用導覽器視圖將產生的 .bindings 檔從 YourPatternInstanceName_EventFilter 專案複製到 IBM Integration Bus 機器。
    2. 在產生的訊息流程中，導覽至「事件訂閱者 JMS 輸入節點」內容。 在「JMS 連線」標籤上，將位置 JNDI 連結內容更新為 .bindings 檔的位置。
  3. 建立佇列管理程式，指定用於過濾事件的佇列，以及指定用於錯誤處理的佇列（如尚未存在）。
  4. 將產生的應用程式 YourPatternInstanceName_EventFilter 部署至整合節點。

如果在配置型樣實例時接受參數的預設值，則可以使用下列步驟來測試產生的應用程式：

  1. 在支援 HTML5 的瀏覽器中，從「應用程式開發」導覽器，開啟獨立資源 > YourPatternInstanceName_EventFilter_client > PacemakerClient.html。
  2. 在瀏覽器頁面中，按一下連接以連接 IBM MessageSight 端點。
  3. 傳送警示事件訊息給 IBM Integration Bus，方法是從事件類型欄位選取警示，然後按一下發佈。
  4. 重複前幾個步驟數次，但從事件類型欄位選取不同的值。
  5. 檢查 MQ 佇列，以確保佇列中有警示事件訊息（且只有警示事件訊息）。
