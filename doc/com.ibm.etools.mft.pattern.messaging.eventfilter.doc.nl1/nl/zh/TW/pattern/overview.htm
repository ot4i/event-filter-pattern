IBM MessageSight Inbound Event Filter 型樣

此型樣使用 IBM Integration Bus 來透過 IBM MessageSight 從行動式用戶端接收事件，基於指定的準則過濾事件，並將符合準則的事件轉遞給後端系統。

當滿足下列條件時使用此型樣：

  • 有大量行動式用戶端透過 IBM MessageSight 發佈訊息。
  • 只有一小部分已發佈的訊息需要後端系統處理。

解決方案

如果大量用戶端透過 IBM MessageSight 發佈訊息，則每個訊息的商業處理可能令後端系統超載。透過使用 IBM MessageSight Inbound Event Filter 型樣，只有符合指定準則的訊息才會轉遞給後端系統，以進行商業處理。不符合準則的訊息可以記載或捨棄。

例如，醫療裝置（如起博器監視器）可能連續發佈有關病患狀態與裝置的事件訊息。包含狀態更新的事件訊息可以記載，包含高嚴重性警示的事件訊息可以轉遞給商業處理系統。
