生成 IBM MessageSight Inbound Event Filter 模式之后要完成的任务

生成该模式之后必须完成的任务。

生成 IBM MessageSight Inbound Event Filter 模式实例之后，完成下列步骤：

  1. 打开 eventFilter 消息流，找到为了配置 IBM MessageSight 端点必须运行的已生成命令。在 IBM MessageSight 机器上运行这些命令。
  2. 可选：如果要在远程集成节点（代理）上运行生成的应用程序，完成下列步骤：
    1. 使用导航器视图，将生成的 .bindings 文件从 YourPatternInstanceName_EventFilter 项目复制到 IBM Integration Bus 机器。
    2. 在生成的消息流中，导航至事件订户 JMS 输入节点属性。在“JMS 连接”选项卡上，用 .bindings 文件的位置更新位置 JNDI 绑定属性。
  3. 创建队列管理器、您指定用于所过滤事件的队列和您指定用于错误处理的队列（如果它们尚不存在）。
  4. 将生成的应用程序 YourPatternInstanceName_EventFilter 部署到集成节点。

如果在配置模式实例时接受了参数的缺省值，可以使用下列步骤来测试生成的应用程序：

  1. 从应用程序开发导航器，在启用 HTML5 的浏览器中打开独立资源 > YourPatternInstanceName_EventFilter_client > PacemakerClient.html。
  2. 在浏览器页面中，单击连接以连接到 IBM MessageSight 端点。
  3. 从事件类型字段选择警报并单击发布，以向 IBM Integration Bus 发送警报事件消息。
  4. 将上一步重复若干次，但是从事件类型字段选择不同的值。
  5. 检查 MQ 队列以确保警报事件消息（且只有警报事件消息）在队列中。
