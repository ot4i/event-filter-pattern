IBM MessageSight Inbound Event Filter 模式

此模式使用 IBM Integration Bus 从移动式客户机通过 IBM MessageSight 接收事件，根据指定条件过滤事件，并将符合条件的事件转发到后端系统。

在符合以下条件的情况下使用此模式：

  • 大量移动式客户机通过 IBM MessageSight 发布消息。
  • 发布的消息只有一部分需要后端系统处理。

解决方案

如果有大量客户机通过 IBM MessageSight 发布消息，为每条消息都启用业务处理可能使后端系统超负荷。如果使用 IBM MessageSight Inbound Event Filter 模式，就只有符合指定条件的消息会转发到后端系统进行业务处理。可以记录或废弃不符合条件的消息。

例如，医学设备（起搏器监测仪等）可能不断发布关于患者和设备的状态的事件消息。可以记录包含常规状态更新的事件消息，并将包含高严重性警报的事件消息转发到业务处理系统。
