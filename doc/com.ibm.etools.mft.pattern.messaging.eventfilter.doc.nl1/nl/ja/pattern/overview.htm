「IBM MessageSight インバウンド・イベント・フィルター」パターン

このパターンは、IBM Integration Bus を使用することによって、IBM MessageSight を介してモバイル・クライアントからイベントを受信し、指定された基準に基づいてイベントをフィルターに掛けて、基準に一致するイベントをバックエンド・システムに転送します。

以下の条件に該当する場合は、このパターンを使用します。

  • IBM MessageSight を介して多数のモバイル・クライアントがメッセージをパブリッシュする。
  • パブリッシュされるメッセージのサブセットだけが、バックエンド・システムで処理される必要がある。

ソリューション

多数のクライアントが IBM MessageSight を介してメッセージをパブリッシュする場合、すべてのメッセージに対するビジネス処理を有効にすると、バックエンド・システムが過負荷になる可能性があります。 「IBM MessageSight インバウンド・イベント・フィルター」パターンを使用すると、指定された基準に一致するメッセージだけがバックエンド・システムに転送され、ビジネス処理されます。 基準に一致しないメッセージについては、ログに記録するか、破棄することができます。

例えば医療機器 (ペースメーカー・モニターなど) は、患者や装置の状態に関するイベント・メッセージを常にパブリッシュし続ける可能性があります。 状態の正常な更新が入ったイベント・メッセージはログに記録し、重大度の高いアラートが入っているイベント・メッセージをビジネス処理システムに転送することができます。
