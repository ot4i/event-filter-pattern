「IBM MessageSight インバウンド・イベント・フィルター」パターンを生成した後に完了するタスク

パターンを生成した後に完了する必要があるタスクです。

「IBM MessageSight インバウンド・イベント・フィルター」パターン・インスタンスを生成した後、以下のステップを実行します。

  1. eventFilter メッセージ・フローを開いて、IBM MessageSight エンドポイントを構成するために実行する必要のある生成済みコマンドを識別します。 IBM MessageSight マシン上でこれらのコマンドを実行してください。
  2. オプション: 生成済みアプリケーションをリモート統合ノード (ブローカー) 上で実行する場合、以下のステップを実行します。
    1. ナビゲーター・ビューを使用して、生成済みの .bindings ファイルを YourPatternInstanceName_EventFilter プロジェクトから IBM Integration Bus マシンにコピーします。
    2. 生成済みメッセージ・フローの中で、イベント・サブスクライバー JMS Input ノード・プロパティーまでナビゲートします。 JMS 接続タブで、「ロケーション JNDI バインディング」プロパティーを .bindings ファイルの場所に基づいて更新します。
  3. キュー・マネージャー、フィルター対象イベント用に指定したキュー、およびエラー処理用に指定したキューを作成します (これらがまだ存在しない場合)。
  4. 生成済みアプリケーション YourPatternInstanceName_EventFilter を統合ノードにデプロイします。

パターン・インスタンスを構成したときにパラメーターのデフォルト値を受け入れた場合、以下のステップに従って生成済みアプリケーションをテストできます。

  1. アプリケーション開発ナビゲーターから、「独立リソース」> YourPatternInstanceName_EventFilter_client > PacemakerClient.html を HTML5 対応ブラウザーで開きます。
  2. ブラウザー・ページで「接続」をクリックして、IBM MessageSight エンドポイントに接続します。
  3. 「イベント・タイプ」フィールドから「アラート」を選択して「パブリッシュ」をクリックすることにより、アラート・イベント・メッセージを IBM Integration Bus に送ります。
  4. 上記の手順を何度か繰り返します。ただし、「イベント・タイプ」フィールドで異なる値を選択します。
  5. MQ キューを調べて、キューの中にアラート・イベント・メッセージが存在する (かつ、アラート・イベント・メッセージ以外は存在しない) ことを確認します。
