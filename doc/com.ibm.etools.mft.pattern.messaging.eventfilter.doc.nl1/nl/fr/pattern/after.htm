Tâches à effectuer après avoir généré le modèle Filtre d'événement entrant IBM MessageSight

Tâches que vous devez exécuter après avoir généré le modèle.

Après avoir généré l'instance de modèle Filtre d'événement entrant IBM MessageSight, procédez comme suit :

  1. Ouvrez le flux de messages eventFilter et identifiez les commandes générées que vous devez exécuter pour configurer le noeud final IBM MessageSight. Exécutez ces commandes sur votre machine IBM MessageSight.
  2. Facultatif : Si vous souhaitez exécuter l'application générée sur un noeud d'intégration distant (courtier), procédez comme suit :
    1. A l'aide de la vue Navigateur, copiez le fichier .bindings généré du projet YourPatternInstanceName_EventFilter vers votre machine IBM Integration Bus.
    2. Dans le flux de messages généré, accédez aux propriétés du noeud Event Subscriber JMS Input. Dans l'onglet Connexion JMS, mettez à jour la propriété Liaisons JNDI d'emplacement avec l'emplacement du fichier .bindings.
  3. Créez le gestionnaire de files d'attente, la file d'attente spécifiée pour les événements filtrés et la file d'attente spécifiée pour le traitement d'erreurs (si elles n'existent pas déjà).
  4. Déployez l'application générée YourPatternInstanceName_EventFilter sur votre noeud d'intégration.

Si vous avez accepté les valeurs par défaut pour les paramètres lors de la configuration de l'instance de modèle, vous pouvez utiliser les étapes suivantes pour tester l'application générée :

  1. Dans le navigateur de développement d'application, ouvrez Ressources indépendantes > YourPatternInstanceName_EventFilter_client > PacemakerClient.html dans un navigateur compatible HTML5.
  2. Sur la page du navigateur, cliquez sur Connexion pour vous connecter au noeud final IBM MessageSight.
  3. Envoyez un message d'événement d'alerte à IBM Integration Bus en sélectionnant Alerte dans la zone Type d'événement et en cliquant sur Publish.
  4. Répétez l'étape précédente plusieurs fois, mais en sélectionnant différentes valeurs dans la zone Type d'événement.
  5. Consultez la file d'attente MQ afin de vous assurer que les messages d'événement d'alerte (et uniquement ces messages) sont dans la file d'attente.
