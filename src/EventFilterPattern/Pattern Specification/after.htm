Tasks to complete after generating the IBM MessageSight Inbound Event Filter pattern

Tasks that you must complete after you have generated the pattern.

If you plan to run the generated application on a remote integration node:

  • Using the navigator view, copy the generated .bindings file from the <pattern_name>_EventFilter project to your IBM Integration Bus machine.
  • In the generated message flow, navigate to the Event Subscriber JMS Input node properties. On the JMS Connection tab, update the Location JNDI Bindings property to the location of the .bindings file

In all cases

  • Deploy the generated application <pattern_name>_EventFilter to your integration node.
  • From the <pattern_name>_EventFilter_client project open PacemakerClient.html in an HTML5 enabled browser.
  • From the browser connect the MessageSight Endpoint and publish different event types,
  • Check the log file and MQ queue for the filtered and unfiltered events.
