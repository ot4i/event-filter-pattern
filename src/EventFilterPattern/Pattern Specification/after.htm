Tasks to complete after you generate the IBM MessageSight Inbound Event Filter pattern

Tasks that you must complete after you have generated the pattern.

After you generate the IBM MessageSight Inbound Event Filter pattern instance, complete the following steps:

  1. Open the eventFilter message flow and identify the generated commands that you must run to configure the IBM MessageSight endpoint. Run these commands on your IBM MessageSight machine.
  2. Optional: If you want to run the generated application on a remote integration node (broker), complete the following steps:
    1. Using the navigator view, copy the generated .bindings file from the YourPatternInstanceName_EventFilter project to your IBM Integration Bus machine.
    2. In the generated message flow, navigate to the Event Subscriber JMS Input node properties. On the JMS Connection tab, update the Location JNDI Bindings property with the location of the .bindings file.
  3. Create the queue manager, the queue that you specified for filtered events, and the queue that you specified for error handling (if these do not already exist).
  4. Deploy the generated application YourPatternInstanceName_EventFilter to your integration node.

If you accepted the default values for the parameters when you configured the pattern instance, you can use the following steps to test the generated application:

  1. From the Application Development navigator, open Independent Resources > YourPatternInstanceName_EventFilter_client > PacemakerClient.html in an HTML5-enabled browser.
  2. In the browser page, click Connect to connect to the IBM MessageSight endpoint.
  3. Send an alert event message to IBM Integration Bus by selecting Alert from the Event Type field and clicking Publish.
  4. Repeat the previous step several times, but select different values from the Event Type field.
  5. Check the MQ queue to ensure that the alert event messages (and only the alert event messages) are in the queue.
